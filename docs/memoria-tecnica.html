1. Resumen del Proyecto

Este proyecto consiste en el diseño e implementación de un brazo robótico articulado de 6 grados de libertad (5 dedos + muñeca) controlado mediante visión por computador en tiempo real.

El sistema captura los movimientos de la mano del usuario a través de una cámara web convencional, los procesa con MediaPipe.js en el navegador, y envía las posiciones por WebSocket a un servidor Flask que corre en un Arduino UNO Q. Este se comunica por USB OTG con un Arduino Mega 2560 que genera las señales PWM para los 6 servomotores MG996R. 

👋 Mano real
   ↓  (cámara web 640×480, ~30 FPS)
🧠 MediaPipe.js (21 landmarks 3D + handedness)
   ↓  (WebSocket seguro wss://)
🐍 Flask + PoseMapper (landmarks → 6 PWMs)
   ↓  (TCP:7500 — SOCAT bridge)
🔗 USB OTG (/dev/ttyACM0, 115200 baud)
   ↓  (comandos F<idx> <pwm_us>\n)
🔌 Arduino Mega 2560 (PWM out D7-D2)
   ↓
⚙️ Servos MG996R ×6 → 🤖 Brazo robótico
🌐 Acceso remoto: El sistema es accesible desde cualquier lugar mediante Cloudflare Tunnel en https://brazo.nxserve.org. No necesita abrir puertos en el router ni conocer la IP local.

3. PoseMapper — El Cerebro

El PoseMapper es el módulo central del sistema (archivo pose_mapper.py, ~700 líneas). Convierte los 21 landmarks 3D de MediaPipe en 6 valores PWM para los servos.

3.1 Pipeline por frame

landmarks (21×3D) + wrist_angle + handedness
    ↓  1. Validación (21 landmarks con x, y, z)
    ↓  2. Cálculo PWM por dedo
    │      └─ Thumb: comparación eje X (TIP vs IP)
    │      └─ Dedos largos: ángulo 3D en articulación PIP
    │      └─ Wrist: detección de rotación por profundidad Z
    ↓  3. Deadband (ignora cambios < 15 µs → evita micro-oscilaciones)
    ↓  4. Rate limit (máx ±60 µs/frame → protección mecánica)
    ↓  5. Clamp a límites seguros por dedo
    →  [thumb, index, middle, ring, pinky, wrist] en µs

3.2 Algoritmo de dedos largos

Para los dedos índice, corazón, anular y meñique se calcula el ángulo 3D en la articulación PIP:

flexion = joint_angle_3D(landmarks, MCP, PIP, TIP)
         ↓
raw = (180° - flexion) × sensibilidad × 1.5
         ↓
PWM = lerp(raw, open_pwm, closed_pwm)

Un dedo recto da ~180° de ángulo → raw bajo → PWM de abierto. Un dedo flexionado da ~0° → raw alto → PWM de cerrado.

3.3 Pulgar

El pulgar usa una estrategia diferente basada en la diferencia de coordenada X entre la punta (TIP) y la articulación media (IP):

x_diff = thumb_tip.x - thumb_ip.x
         ↓
normalized = clamp((x_diff + 0.01) / 0.02, 0, 1)
         ↓
PWM = closed_pwm + (open_pwm - closed_pwm) × normalized

El parámetro handedness ('Left' o 'Right') invierte la lógica según la mano detectada.

3.4 Muñeca (rotación)

La muñeca detecta rotación de la mano (no flexión vertical) mediante la diferencia de profundidad Z entre los MCP del índice y el meñique:

z_diff = landmarks[INDEX_MCP].z - landmarks[PINKY_MCP].z
         ↓
wrist_raw = clamp((z_diff + 0.04) / 0.08, 0, 1)
         ↓
PWM = closed_pwm + (open_pwm - closed_pwm) × wrist_raw

Incluye auto-calibración en vivo: observa el rango mínimo/máximo real y lo estira al rango completo.

4. Firmware del Mega 2560

El firmware (mega_servos.ino, 453 líneas) se encarga de recibir comandos por USB, mover los servos con suavidad y vigilar que la comunicación no se caiga.

4.1 Protocolo serie

ComandoFormatoEjemploDescripción
FingerF<idx> <pwm_us>\nF0 1500\nPosiciona servo <idx> al ancho de pulso <pwm_us>
HeartbeatH\nH\nKeep‑alive cada 500ms desde Flask
ACKOK\nOK\nRespuesta del Mega a un Heartbeat
DebugD\nD\nToggle modo debug por USB

4.2 Parser por máquina de estados

El parser usa un autómata de 3 estados sin memoria dinámica (buffer estático de 16 bytes):

WAITING_F  → esperando 'F' (comando) o 'H' (heartbeat)
READY_IDX  → 'F' recibido, esperando dígito del índice (0-5)
READY_PWM  → índice recibido, acumulando dígitos del PWM

El heartbeat (H) se procesa en cualquier estado para garantizar robustez.

4.3 Interpolación trapezoidal

El movimiento de los servos se suaviza con un perfil de velocidad trapezoidal:

Velocidad (µs/paso)
    ↑
MAX ──────╱╲──────────    ← Crucero (8 µs/paso)
   ╱╲    ╱  ╲    ╱╲
  ╱  ╲  ╱    ╲  ╱  ╲
 ╱    ╲╱      ╲╱    ╲
─┴──────┴──────┴──────┴──→ Distancia
  Acel.   Crucero   Decel.

La distancia de frenado se calcula con d = v² / (2·a), donde v es la velocidad actual y a = 2 µs/paso².

4.4 Watchdog de seguridad

Si no se recibe ningún comando (F o H) durante 2500 ms, el Mega lleva todos los servos a safe pose (1300 µs) automáticamente. Esto protege el brazo si el UNO Q se cuelga o se pierde la comunicación.

5.3 Aislamiento eléctrico

El UNO Q (cerebro Linux) está aislado eléctricamente del Mega (potencia). Los picos de corriente de los servos (hasta 2 A en stall) nunca afectan al Linux. La comunicación es por USB OTG, lo que proporciona aislamiento galvánico parcial.

8. Instalación Rápida

# Requisitos: Node.js ≥ 18, Python ≥ 3.10, Arduino IDE 2.0+

# 1. Clonar y construir frontend
git clone <repo> && cd BrazoRoboticoV2
cd frontend && npm install && npm run build && cd ..

# 2. Instalar backend (en UNO Q)
cd backend
pip install -r ../deploy/requirements.txt

# 3. Servicios systemd (en UNO Q)
sudo cp deploy/systemd/socat-mega.service /etc/systemd/system/
sudo cp deploy/systemd/robot-hand.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now socat-mega.service robot-hand.service

9. Conclusión

El Brazo Robótico V2 demuestra cómo integrar tecnologías web modernas (React, MediaPipe, WebSocket) con sistemas embebidos (Arduino UNO Q, Mega 2560) para crear un sistema de control gestual en tiempo real.

La arquitectura elegida —con el UNO Q como cerebro Linux y el Mega como controlador PWM— proporciona un equilibrio óptimo entre capacidad de procesamiento y control de hardware. La migración a USB OTG simplificó significativamente la comunicación, eliminando la dependencia del STM32 como bridge intermedio.

El sistema es accesible desde cualquier lugar gracias al túnel Cloudflare, permitiendo demostraciones remotas sin necesidad de configurar redes complejas.

✅ Estado actual: Sistema completamente operativo. Todos los dedos calibrados físicamente. Demo en vivo: https://brazo.nxserve.org

Mejoras futuras

  • Calibración de muñeca para que el mapeo rotacional coincida con el rango físico.
  • Implementar grabación y reproducción de secuencias (ya incluido en el frontend).
  • Migrar Flask a un servidor WSGI de producción (Waitress/Gunicorn).
  • Añadir certificados SSL para HTTPS local (actualmente solo por Cloudflare).
  • Tests unitarios para pose_mapper.py (el módulo core no tiene tests).
